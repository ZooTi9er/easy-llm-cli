# easy-llm-cli 自定义模型 API 调用栈分析需求文档

## 项目背景

easy-llm-cli 是一个基于 TypeScript 的命令行 LLM 交互工具，支持多种 LLM 提供商，包括 Google Gemini 和自定义 OpenAI 兼容 API。本项目需要深入分析其自定义模型的 API 调用栈，并实际测试自定义 OpenAI 兼容 API 的集成。

## 分析目标

### 主要目标
1. **详细分析 API 调用栈流程**：从用户输入到 API 响应的完整调用链路
2. **测试自定义 OpenAI 兼容 API**：验证环境变量配置和实际调用
3. **识别关键配置点**：理解环境变量如何影响 API 调用行为
4. **验证错误处理**：测试 API 调用失败时的处理机制
5. **性能分析**：评估 API 调用的性能特征

### 测试环境配置
- **API 基础URL**: http://mini.ewuzhe.dpdns.org:2000/v1
- **API 密钥**: sk-wuzhe12345
- **模型提供商**: openai/gemini-2.5-flash
- **环境变量**: 
  - OPENAI_API_BASE
  - OPENAI_API_KEY
  - LLM_PROVIDER

## 技术分析范围

### 1. 核心配置文件分析
- `packages/core/src/config/config.ts`: 主配置类
- `packages/core/src/custom_llm/`: 自定义 LLM 集成模块

- 认证类型判断

#### 2.3 LLM 抽象层
- ContentGenerator 接口实现
- CustomLLMContentGenerator 类
- ModelConverter 转换器

#### 2.4 API 调用层
- OpenAI SDK 调用
- 请求格式转换
- 响应格式转换
- 流式处理

#### 2.5 错误处理层
- API 错误捕获
- 重试机制
- 用户反馈

### 3. 关键代码路径识别

#### 3.1 环境变量处理路径
```typescript
// 需要分析的代码路径
process.env.OPENAI_API_BASE -> Config 类 -> CustomLLMContentGenerator
process.env.OPENAI_API_KEY -> OpenAI SDK 配置
process.env.LLM_PROVIDER -> 认证类型判断
```

#### 3.2 API 调用路径
```typescript
用户输入 -> GeminiClient -> ContentGenerator -> CustomLLMContentGenerator -> 
OpenAI SDK -> 自定义 API -> 响应转换 -> UI 显示
```
